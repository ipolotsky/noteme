# Issue #14: Beautiful date cards with AI wish selection

## What was done

Replaced the multi-message feed (list of beautiful dates with pagination) with a single-card gallery view. Each card shows one beautiful date with full details and navigation buttons.

### Key changes

1. **Feed handler rewrite** (`app/handlers/feed.py`):
   - Removed `_send_feed` (sent multiple messages per page with PAGE_SIZE=5)
   - Added `_build_card` - builds a single rich card with date label, calendar date, relative time, and AI-selected wish
   - Added `_select_best_wish` - uses GPT-4o-mini to pick the most relevant wish from user's notes matching event tags
   - Added `_show_card_new` - sends one card message and stores its ID in FSM
   - Navigation via `edit_text` on prev/next (no message deletion needed)
   - New handlers: `feed_card` (navigation), `feed_wishes` (show all wishes), `feed_share` (generate share link)

2. **Feed keyboard** (`app/keyboards/feed.py`):
   - Replaced `feed_list_kb` and `feed_item_kb` with `feed_card_kb`
   - Card buttons: prev/next navigation, "To event" + "All wishes", "Share", "Back"

3. **i18n** (`app/i18n/en.json`, `app/i18n/ru.json`):
   - Added keys: `feed.wish`, `feed.counter`, `feed.next`, `feed.prev`, `feed.all_wishes`, `feed.wishes_title`, `feed.no_wishes`
   - Changed `feed.title` from "Beautiful Dates Feed" to "Beautiful Dates"

4. **Tests** (`tests/test_unit.py`):
   - Updated `TestFeedKeyboard` tests for new `feed_card_kb` function
   - Added `event_id` to `_make_bd_obj` helper

## Key decisions

- AI wish selection uses GPT-4o-mini with temperature=0, max_tokens=10 (just returns a number)
- Falls back to first note if no OpenAI key or on any error
- Wishes found by matching event tags to note tags (existing pattern)
- Navigation uses `edit_text` for smooth UX (no message flicker)
- FSM still stores feed message IDs for cleanup when re-entering feed

## Files changed

- `app/handlers/feed.py` - complete rewrite from list to gallery
- `app/keyboards/feed.py` - replaced list/item keyboards with card keyboard
- `app/i18n/en.json` - new feed keys
- `app/i18n/ru.json` - new feed keys
- `tests/test_unit.py` - updated feed keyboard tests
- `.gitignore` - agent runner files

## Uncertainties

- `feed_card_kb` in `keyboards/feed.py` duplicates keyboard logic from `_build_card` in `handlers/feed.py` - the keyboard file function is not actually used anywhere in handlers
- AI wish selection prompt is in English only, may work less well for Russian content
